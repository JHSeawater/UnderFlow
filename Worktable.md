# UNDERFLOW 기능 구현 명세서

팀 회의 및 역할 분배를 위해 `GDD.md` 및 `Task.md`를 바탕으로 구현해야 할 모든 기능들을 시스템 콜이나 내부 함수명 없이, '기능' 단위로 빠짐없이 분류한 명세입니다.

## 1. 계정 및 인증 시스템 (Account & Auth)
- **단일 Key 기반 인증**: 복잡한 가입 절차나 비밀번호 없이 고유 Key(문자열) 하나로 즉시 접속 및 가입 처리
- **유저 정보 영속성(저장) 관리**: 유저의 자본금 및 자산 데이터를 서버 내 단일 파일에 안전하게 저장하고 불러오기
- **유저 데이터베이스 수정 보호**: 서버에서 유저의 자산(DB) 수치가 변경될 때, 다른 작업이 동시에 접근하지 못하도록 레코드 단위의 잠금(Lock)을 걸어 데이터 훼손 방지
- **논리적 계정 소각 (파산 처리)**: 자금을 모두 잃고 파산한 유저의 데이터를 특수 처리하여 해당 Key의 영구 접속 차단
- **이중 접속 차단 및 하트비트 세션 동기화**: 이미 접속 중인 Key의 중복 로그인을 차단하고, 서버 크래시 및 워치독 재기동 시 세션 정합성을 위한 하트비트 신호 송출 및 유실 세션 복구 처리

## 2. 핵심 게임플레이 및 경제 시스템 (Core Gameplay)
- **기밀 문서 자동 주기 스폰**: 비동기 서버 이벤트 엔진에 의해 상점에 주기적으로 기밀 문서를 새롭게 생성하고 브로드캐스트
- **기밀 문서 독점 구매 (`/buy`)**: 상점에 등록된 기밀 문서를 선착순으로 구매하며, 구매 즉시 서버 상점에서 매물 삭제 (단일 독점 매물)
- **속성 태그 부여 시스템**: 모든 기밀 문서에 1~N개의 랜덤 속성 태그(예: `[고객정보]`, `[군사무기]`)를 조합하여 생성
- **NPC 의뢰 스폰 및 생태계 관리**:
  - 특정 속성 태그 조합을 완벽히 요구하는 NPC를 무작위 현상금과 함께 생성
  - 오랫동안 해결되지 않은 악성 NPC 의뢰를 지속적으로 만료(Despawn)시켜 시장 생태계 순환 촉진
- **기밀 문서 매각 및 조합 검증 로직 (`/sell`)**:
  - 유저가 제출한 여러 문서들의 태그 합집합이 NPC가 요구하는 조합을 완전히 포함(부분집합 관계)하는지 검증 (불필요 초과 태그도 소모)
  - 여러 명이 동시에 같은 NPC에게 매각을 시도할 경우(선착순 매각 충돌), 가장 빠른 1명에게만 현상금 지급
- **기밀 문서 강제 파기 (`/dispose`)**: 인벤토리 샌드박스가 꽉 차거나 악성 재고가 생겼을 때, 어떠한 보상이나 환급금 없이 0원에 즉각 파일을 파기(손절매)하여 공간 확보

## 3. 인벤토리 및 파일 시스템 (Inventory System)
- **개인 샌드박스 할당**: 플레이어가 접속하면 본인만의 고유한 샌드박스 디렉토리(폴더) 생성
- **실물 파일 기반 권한 제어**: 기밀 문서를 구매/매각할 때 서버 마스터 폴더와 유저 샌드박스 간에 실제 물리적 파일을 이동하여 소유권 통제
- **인벤토리 용량 제한**: 유저가 동시에 가질 수 있는 기밀 문서 개수를 인당 최대 5개로 엄격히 제한하고 초과 구매 방지
- **보유 아이템 순회 조회 (`/inventory`)**: 유저가 현재 자신의 인벤토리에 보관 중인 파일(기밀 문서) 목록을 읽어와 출력하며, 이때 운영체제 기본 숨김 폴더(., ..)는 파싱 예외 처리하여 노출 차단

## 4. 비동기 이벤트 및 소셜 사보타주 (Events & Sabotage)
- **실시간 전역 채팅 시스템**: 유저들이 서로 소통하며 눈치 싸움이나 정보 교환을 할 수 있는 전체 채팅망 지원
- **글로벌 속보 뉴스 (서버 브로드캐스트)**: 특정 NPC 출몰이나 시장 변화를 예측할 수 있는 사전 힌트/뉴스를 주기적으로 전체 유저에게 전송
- **공중파 유출 (자산 동결 및 존버 메타)**:
  - 언론 유출 뉴스가 터지면 특정 태그를 가진 기밀의 가치를 일정 시간(3~5분) 동안 '0원 폭락 및 동결' 처리 (NPC 판매 불가, 시장 스폰 중지)
  - 동결 시간이 지나면 뉴스가 잠잠해지며 기밀의 가치를 원래대로(1.0배) 복구(재상장)하여 버틴 유저에게 보상
- **시각적 글리치 공작 (`/rumor`)**:
  - 막대한 수수료를 내면 타겟 유저의 화면 텍스트(가격, 태그 등)가 일시적으로 깨지거나 뒤섞이는 시각적 방해 효과 발동
  - 무분별한 남용을 막기 위해 서버 내에 공유되는 쿨타임(대기시간) 적용

## 5. 긴급 파기 미니게임 (Panic Escape Sequence)
- **경찰 레이드 이벤트 발동**: 경찰 추적망 활성화 알림(패킷)을 서버 전체에 긴급 전송
- **위기 경고 시각화**: 클라이언트의 화면 전체가 붉은색으로 점멸하며 긴장감 조성
- **긴급 회피(단축키) 하이재킹**: 유저가 위험을 감지하고 긴급 정지 단축키(Ctrl+C)를 눌렀을 때, 프로그램이 바로 종료되는 대신 채팅/마켓을 강제 잠금하고 방탈출 모드로 전환 (비상시 강제 안전 탈출을 위한 Ctrl+\\ 우회 매핑 제공)
- **암호 타이핑 방탈출**: 제한 시간 내에 화면에 뜬 암구호를 오타 없이 타이핑하는 미니게임 구동 (시간 초과 실패 시 서버에서 강제 퇴출)

## 6. 승리 및 게임 오버 조건 (Win & Game Over)
- **목표 금액 달성 및 자금 전송**: 서버 내 유저들의 보유 자산을 실시간 모니터링하여 목표 할당량 도달 즉시 전 재산을 익명으로 강제 전송(초기화)
- **승리자 엔딩 처리**: 가장 먼저 목표액을 달성한 1인의 화면에 승리 메시지("모든 빚을 갚았군. 약속대로 넌 이제 자유다.")를 띄우고 게임에서 명예롭게 퇴장
- **서버 내 연쇄 게임 오버 및 정산 (Post-Game Lobby)**: 승리자가 발생하는 즉시, 전역 거래 동결(블랙아웃) 및 최종 랭킹 스코어보드를 노출하고 **30초 에필로그 카운트다운** 동안 소통 시간을 부여한 뒤 안전 정상 종료
- **파산 즉각 처형 (Permadeath)**: 자금을 모두 잃고 빚을 진 유저는 즉시 서버에서 강제 퇴출되며, 해당 계정 데이터는 영구적으로 삭제되어 처음부터 재시작 필요

## 7. 시스템 및 네트워크 아키텍처 (Network & Server)
- **다중 접속 네트워크 서버**: 여러 클라이언트가 독립적으로 동시 접속하여 충돌 없이 플레이할 수 있는 백엔드 서버 구축
- **무중단 서버 모니터링 (워치독) 및 세션 리셋**: 게임 서버(자식)가 다운되면 감시 시스템(부모)이 즉각 재가동하며, 하트비트 세션 상태 및 active_keys 정합성을 복구하는 방호 체계
- **데이터 패킷 직렬화 통신 규약**: 서로 다른 PC나 CPU 간에 통신할 때 데이터가 깨지거나 왜곡되지 않도록 네트워크 표준 규격으로 데이터 구조화
- **동시성 충돌 방지 및 데드락(교착상태) 차단**: 
  - 다수의 유저가 동시에 같은 아이템을 매각하거나 접근할 때, 락(Lock) 순서를 정렬하는 오름차순 알고리즘을 통해 서버가 영원히 멈추는 데드락 현상을 원천 차단
- **파일 이동 취약점 방어**: 상점과 유저 샌드박스의 최상위 경로를 묶어 운영체제 간 파일 이동 시 발생하는 오류(EXDEV) 선제적 차단
- **안전한 클라이언트 연결 관리**: 클라이언트가 비정상적으로 종료되었을 때, 연결을 안전하게 해제하고 접속자 목록을 정리

## 8. 클라이언트 UI 및 사용자 경험 (Client UI/UX)
- **최소 해상도 런타임 방어막**: 게임 실행 시 터미널 크기가 최소 요구 해상도(100x30)에 미달하면 화면이 깨지는 것을 막기 위해 친절한 에러 문구 출력 후 안전하게 실행 차단
- **터미널 동적 리사이징 연출**: 게임 도중 유저가 터미널 창 크기를 드래그해 변경하더라도 화면 요소가 깨지지 않고 자동으로 조절되어 다시 그려지는 기능
- **비율 기반 분할과 잔여 공간 보정**: 실수 연산 시 발생하는 1~2칸의 미세한 검은 여백을 남은 잔여 크기로 재계산하여 화면 가장자리 빈틈을 없애는 꽉 찬 화면 렌더링
- **입력창 분리 및 버퍼 오버플로우 방어**:
  - 채팅이 실시간으로 올라와도 입력 중인 커서가 위로 밀려 올라가지 않도록, 채팅/로그 출력 창과 프롬프트 입력 창을 물리적으로 분리
  - 지정된 가용 너비 이상 타자를 칠 수 없도록 입력을 무시하여 메모리가 초과되는 오버플로우 방지
- **비동기 실시간 입력 엔진**: 유저가 명령어 타자를 치고 있는 도중에도 서버에서 날아오는 속보, 채팅, NPC 스폰 메시지 등을 끊김 없이 실시간으로 화면 윗단에 갱신
- **강제 커서 원위치 복구**: 실시간 메시지가 화면에 업데이트될 때 프롬프트 입력창의 커서가 위로 튕겨버리는 터미널 고질적 버그를 방어하여 원래 위치로 강제 고정
- **UI 환경 강제 복구 (Fail-Safe)**: 비정상적인 종료나 오류로 튕기더라도 유저의 기존 터미널 화면 설정이 망가지지 않도록 원래대로 깔끔하게 복구
- **시각화 디테일 강화**: 모든 서버 알림에 시스템 시간(타임스탬프)을 남기고, 화면 우측 끝에 '퀵 헬프(단축키 매뉴얼)' 렌더링

## 9. 프로젝트 통합 빌드 환경 (Project Common)
- **통합 빌드 스크립트 작성**: 한글 폰트 깨짐 방지와 멀티스레딩 지원이 포함된 전용 빌드 파일(`Makefile`) 구성
- **메모리 누수 검증 아키텍처**: 개발 마지막 단계에서 런타임 중 누수되는 메모리 찌꺼기가 없는지 교차 검증하는 환경 세팅

## 10. 🎯 팀원 역할 분배 추천 및 업무 밸런싱 (Role Assignment)
팀원 간 불만이 생기지 않도록 각 기능의 **개발 난이도**와 **작업 성향(네트워크/시스템/UI)**을 종합적으로 고려하여 3개의 독립적인 역할로 분배했습니다. 각 영역이 철저히 분리되어 있어 소스 코드 충돌 없이 병렬 개발이 가능합니다.

### 🧑‍💻 [역할 A] 서버 아키텍처 및 이벤트 엔진 (Network & Background)
**성향:** 네트워크 통신 흐름, 타이머 제어, 패킷 설계에 관심이 많은 팀원
**난이도:** ⭐⭐⭐⭐ (상) - 다중 클라이언트 통신과 비동기 백그라운드 처리가 핵심입니다.
- **[4. 이벤트]** 글로벌 속보 뉴스 브로드캐스트, 공중파 유출(동결/복구 타이머), 실시간 채팅망, **시장 매물 스폰 시 파일 동시 생성, 사보타주 글리치 타겟 단일 전송(Unicast)**
- **[7. 시스템]** 다중 접속 소켓 서버 뼈대 구축, 워치독 모니터링 체계, 데이터 패킷 직렬화 규약
- **[1. 계정]** 접속 패킷 처리 및 메모리 상의 이중 접속 감지/차단, **늦은 접속자(Late Joiner) 초기 마켓 동기화(Unicast) 전송**
- **[6. 게임오버]** 역할 B의 승리자 판별 시그널을 수신하여 전체 유저에게 거래 동결(블랙아웃) 및 정산 랭킹 패킷 브로드캐스트 발송 후 30초 타이머 개시

### 🧑‍💻 [역할 B] 백엔드 코어 로직 및 파일 시스템 (System, FS & DB) - **[PM 추천]**
**성향:** 운영체제(OS) 이해도가 높고, 동시성 오류(데드락)나 파일 잠금 등 까다로운 논리 구조 해결을 즐기는 팀원
**난이도:** ⭐⭐⭐⭐⭐ (최상) - 시스템의 심장이자 크리티컬 에러가 가장 많이 발생하는 곳이므로 책임자인 PM이 맡는 것을 강력히 권장합니다.
