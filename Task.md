  - [ ] 컴파일 시 필수 링킹 옵션(`-pthread`, `-lncurses`)이 포함된 전용 `Makefile` 구축
  - [ ] 프로젝트 빌드 방법, 실행 옵션, 주요 기능이 완벽히 정리된 `README.md` 작성
- [ ] **패킷 프로토콜 엔디안(Endianness) 변환 규약 정립**
  - [ ] `protocol.h` 헤더 작성 시 다중 클라이언트의 아키텍처(CPU) 호환성을 위해 `int` 등 4바이트 이상 데이터에 대해 송신 전 `htonl()`, 수신 후 `ntohl()` 강제 적용 보장

## 📡 User 1: 서버 네트워크 및 이벤트 스레드 담당 (Network & Background)
- [ ] **소켓 통신 뼈대 및 구조체 설계**
  - [ ] `socket()`, `bind()`, `listen()`, `accept()` 기반 포트 개방
  - [ ] Thread-per-client 모델 `pthread_create()` 연동
  - [ ] 로그인, 채팅, 찌라시, 매매 요청 등 패킷(`Packet`) 프로토콜 구조 구조화
  - [ ] **[망실 방어]** 클라이언트 강제 종료 시 `recv() <= 0` 예외를 Catch하여 해당 Socket 안전하게 `close()`
  - [ ] 퇴장 유저를 '현재 접속자 배열'에서 삭제할 때 데이터 충돌을 막기 위한 전역 상태 `Mutex Lock` 적용
- [ ] **좀비 프로세스 방어 파이프라인 (Watchdog)**
  - [ ] 부모 `monitor` 프로세스에서 `fork()` 호출 후 `exec()` (ex: `execv`) 시스템 콜을 통해 자식 서버 교체 실행
  - [ ] 자식 프로세스(서버) 붕괴 시 `waitpid()` 로 백그라운드 추적 및 재가동 리스크 차단
- [ ] **비동기 자동화 이벤트 스레드 (Event Engine)**
  - [ ] `sleep()` 타임 루프를 맴도는 백그라운드 이벤트 전이 전담 스레드 파생
  - [ ] 무작위 속성(태그)을 가진 "시장 매물" 1개씩 스폰 및 전체 브로드캐스트
  - [ ] 여러 속성 조합을 요구하는 "NPC 의뢰" 스폰 및 브로드캐스트
  - [ ] 간헐적으로 특정 태그 가치를 0으로 폭락시켜 3~5분간 NPC 스폰 및 상점 판매를 차단하는 "Public Leak(동결)" 뉴스 브로드캐스트
  - [ ] 동결 쿨다운 시간 종료 시, 해당 기밀의 가치를 다시 1.0배로 복구(재상장)시켜 기다린 유저에게 보상하는 기능 추가
  - [ ] 오래된 NPC 보드 의뢰 소멸(Aging/Despawn) 수거 루프 구축
  - [ ] 경찰 레이드 🚨 경고 패킷 파생 및 방송

## 🔒 User 2: 백엔드 파일 입출력 및 동기화 담당 (System, FS & DB) [최상위 난이도]
- [ ] **유저 DB 및 레코드 락 (File I/O Persistence)**
  - [ ] 접속 유저에게 자본금 할당 및 `users.dat` 파일 추적
  - [ ] `open()`, `lseek()`, `write()` 를 통한 자산 갱신 직접 제어 체계
  - [ ] 타 스레드가 수정 중일 때 보호하는 `fcntl()` 유저 자산 레코드 락 구현
- [ ] **인벤토리 샌드박스와 파일 시스템 디렉토리 연산 (File System)**
  - [ ] 클라이언트 최초 유입 시 `mkdir()` 로 `./data/users/[user_id]/` 하위에 고유 개인 샌드박스 생성
  - [ ] 마스터 상점(`./data/master/`) ↔ 유저 샌드박스 간 파일 이동 시 EXDEV 에러를 피하도록 물리적 경로 고정
  - [ ] 매입/매각 시 `rename()` 호출하여 파일 객체를 원자적으로 은닉 및 `chmod()` 차단
  - [ ] `/dispose` (강제 폐기) 명령 구동 시 `unlink()` 로만 파일 파쇄 (어떠한 수익 환급이나 보상도 없이 0원 처리)
  - [ ] **[샌드박스 순회]** `/inventory` 요청 시, `opendir()`과 `readdir()`를 사용해 해당 유저의 경로 파일들을 순회하여 패킷 전송 (단, `.` 및 `..` 디렉토리 파싱 예외 처리 필수)
- [ ] **이중 동시성 방어 및 교착상태(Deadlock) 원천 차단 알고리즘**
  - [ ] 상점 배열 접근 및 NPC 매각 시 중복 탈취 방지용 `pthread_mutex_lock` 적용
  - [ ] **[핵심]** 파일 번호(ID)가 2개 이상일 경우 반드시 ID 번호 오름차순으로 Lock을 잠구어 Circular Wait 현상 봉쇄

## 🖥️ User 3: 클라이언트 UI 및 시그널 하이재킹 담당 (UI/UX & Signals)
- [ ] **클라이언트 스레드 투트랙 분리 및 수동 버퍼 구현**
  - [ ] 클라이언트 실행 즉시 `pthread_create()` 호출로 **2-Thread Blocking 분리 구조 설계**
    - [ ] `스레드 A (메인)` : `wgetch()` 로 입력 프롬프트 블로킹 무한 대기 및 렌더링
    - [ ] `스레드 B (수신)` : `recv()` 로 서버 패킷 무한 대기 후 수신 시 UI 상단 갱신
  - [ ] `cbreak()`, `noecho()`, `timeout(0)` 환경에서의 비동기 커스텀 `char buffer[]` 제한 입력 구현
- [ ] **Ncurses 6분할 윈도우 인터페이스 (UI Engine)**
  - [ ] `initscr()` 등 호출 직후 `getmaxyx()` 연산으로 최소 해상도(100x30) 미달 시 프로그램 강제 종료 및 에러 문구 렌더링 (Fail-Safe 적용)
  - [ ] 배율(Float) 할당의 정수 버림 오차 방지를 위해 최종단 윈도우는 잔여 길이(`COLS` 혹은 `LINES`에서 누적폭을 뺀 값)로 연산 할당
  - [ ] `wgetch()` 대기 루프 내부에서 창 크기 조절 시스템 인터럽트를 잡아내는 `KEY_RESIZE` 분기문으로 `delwin()` 및 동적 `Redraw` 
  - [ ] **[창 분할 코딩]** 커서 스크롤 오작동 방지를 위해 화면 최하단부를 `출력용 채팅/로그 창(scrollok 활성)`과 `단 1줄 크기의 순수 입력용 윈도우(scrollok 비활성)`로 분리 선언
  - [ ] **[Buffer Overflow 차단]** 순수 입력창의 가용 너비(전체폭에서 퀵헬프 길이를 제외한 범위)를 초과하는 `wgetch()` 타건은 메모리에 담지 않고 무시하도록 블로킹
  - [ ] **[UI 디테일]** 입력 윈도우 로드 시 우측 끝라인에 맞추어 `mvwprintw`를 사용해 '퀵 헬프(단축키 매뉴얼)' 사전 렌더링
  - [ ] **[시스템 타임]** 채팅 로그 `wprintw()` 렌더링 시, 사전에 `<time.h>` 의 `localtime()`을 파싱하여 실시간 타임스탬프 규격 가공
  - [ ] 수신 스레드에 의한 `wrefresh` 갱신 튕김 여파 방어용 `wmove()` 프롬프트 입력 커서 원위치 강제 복귀 루틴
- [ ] **터미널 복구 복원 및 사보타주 연출 (Signals & Fail-Safe)**
  - [ ] **[Fail-Safe 환경 복구]** 비정상 오류나 `SIGINT`로 프로그램 이탈 전, 반드시 `endwin()`을 호출하도록 `atexit` 또는 시그널 핸들러 상단에 방어 코드 결합 (터미널 오작동 원천 차단)
  - [ ] 사보타주 패킷 수신 시, 터미널 색상(`start_color`)과 반전(`A_REVERSE`)을 무작위로 뒤섞는 텍스트 글리치 랜덤 렌더링
  - [ ] 경찰 레이드 수신 중 오직 `Ctrl+C` 에 반응하는 시그널 하드웨어 후킹(`signal(SIGINT)`)
  - [ ] `is_panic = 1;` 변수 감지 시 암호 타이핑 미니게임 렌더링 전환 (제한시간 오버 시 강제 서버 이탈 연동)
