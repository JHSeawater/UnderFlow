오늘(5월 3일)을 기준으로 기말고사(6월 16일)와 첫 발표일(6월 2일)을 고려하여, **5월 말(5월 31일)까지 실질적인 개발을 100% 완료**하고 6월부터는 시험 공부와 발표 준비에 집중할 수 있도록 구성한 4주 완성 현실적 마일스톤입니다.

## 📅 주차별 개발 마일스톤 (4주 완성)

### 🚀 [1주차] 5월 4일 ~ 5월 10일 : 뼈대 구축 및 기반 공사 (Foundation)
**목표:** 각 파트의 독립적인 뼈대를 세우고 클라이언트-서버 간 연결 성공하기
- **[역할 A (네트워크)]**: TCP 다중 접속 서버/클라이언트 소켓 연결 및 종료 뼈대 구현. 채팅 패킷 송수신 테스트 **및 하트비트 세션 복구 및 늦은 접속자(Late Joiner) 마켓 초기 동기화(Unicast) 뼈대 구축**.
- **[역할 B (시스템)]**: `users.dat` 파일 입출력 로직 작성 및 접속 시 개인 샌드박스(디렉토리) 생성 로직 구현.
- **[역할 C (UI)]**: Ncurses 라이브러리를 활용한 6분할 윈도우 레이아웃 뼈대 작성 및 **다중 태그 비트마스크 순회 렌더링 로직 적용**.
- **[공통]**: 5월 4일 첫 회의에서 **통신 패킷 구조체(`struct`) 최우선 확정** (데이터 형식 충돌 방지), `Makefile` 통합 및 Github 레포지토리 세팅.

### ⚔️ [2주차] 5월 11일 ~ 5월 17일 : 핵심 게임플레이 구현 (Core Loop)
**목표:** 아이템이 생성되고, 사고 팔며 인벤토리에 들어오는 핵심 경제 시스템 완성
- **[역할 A]**: 백그라운드 이벤트 스레드 구동 (주기적으로 시장 매물 스폰 시 **물리 파일 생성** 및 NPC 생성 패킷 전체 브로드캐스트).
- **[역할 B]**: `/buy`, `/sell` 부분집합 태그 조합 검증(초과 소모 포함) **및 동결 상태 차단** 로직, **락(Lock) 순서 규약 선제 설계** (데드락 원천 방지), `rename` 실물 파일 이동 및 **존재 여부 확인(`stat`) / 시스템 콜 에러 복구 로직** 구현.
- **[역할 C]**: 비동기 입력 처리(채팅과 명령어 입력을 렉 없이 동시 수행), 퀵 헬프 UI 렌더링, 수신 패킷을 화면에 즉각 업데이트하는 로직 구현.
- **[공통]**: **중간 통합 점검 (5월 17일)** - UI 없이 터미널 로그만으로 '접속 -> 구매 -> 파일 이동' 전체 흐름 빌드 테스트.

### 💣 [3주차] 5월 18일 ~ 5월 24일 : 특수 이벤트 및 사보타주 (Events & Sabotage)
**목표:** 게임의 재미를 끌어올리는 돌발 변수와 방탈출 등 변칙 시스템 결합
- **[역할 A]**: 공중파 유출(자산 동결) 로직, 경찰 레이드 경고 패킷 전송, **사보타주 대상 단일 전송(Unicast)**, **방탈출 성공 응답(`PKT_RES_MINIGAME_OK`) 구현**, **블랙아웃 및 30초 정산 카운트다운 브로드캐스트 전송**.
- **[역할 B]**: 악성 재고 파기(`/dispose`), **`/rumor` 수수료 차감 로직**, 2주차 락 체인 기반 데드락 방어 검증, 유저 자산 갱신 시 **목표액 도달(승리) 감지 및 블랙아웃 트리거 전달** 및 파산 유저 **논리적 계정 소각** 로직 구현.
- **[역할 C]**: 시그널(Ctrl+C) 하이재킹 및 Ctrl+\\ 비상 탈출 적용, 방탈출 미니게임 UI, **성공 응답 수신 시 잠금 해제 복구 로직**, 상대방 화면 글리치(`/rumor`) 연출 **및 정산 랭킹/30초 에필로그 후 안전 종료** 구현.
- **[공통]**: **데모 시나리오 선정** - 최종 발표(6/2)에서 보여줄 시연 시나리오를 미리 짜두어 4주차 QA 포커스를 명확히 하기.

### 🛡️ [4주차] 5월 25일 ~ 5월 31일 : 통합, 최적화 및 QA (Integration & Polish)
**목표:** 개별 개발된 코드를 하나로 합치고 극한의 버그 테스트 진행 (기말고사 전 개발 종료)
- **[전원 참석]**: 모든 파트의 코드를 합치고 실제 터미널 3~4개를 띄워 멀티플레이 플레이테스트.
- **[역할 A]**: 부모 프로세스를 활용한 워치독(서버 튕기면 자동 재시작) 체계 최종 점검.
- **[역할 B]**: 동시에 여러 명이 하나의 NPC에 `/sell`을 연타했을 때 서버가 뻗지 않는지(데드락) 집중 방어 테스트.
- **[역할 C]**: UI 레이아웃 깨짐(Fail-Safe) 방어, 윈도우 크기를 마구 줄였을 때 튕기는지 버그 잡기.
- **[공통]**: `Valgrind` 구동 시 Ncurses 고유 누수 오진을 고려하여, **우리가 직접 할당(`malloc`)한 메모리에 대한 해제 확인**을 최종 목표로 설정.
