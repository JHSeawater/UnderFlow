# 시스템 프로그래밍 팀프로젝트 - UNDERFLOW

## 1. 게임 개요 (Game Overview)
- **게임명**: UNDERFLOW
- **장르**: 터미널 기반 멀티플레이어 경제/소셜 시뮬레이션
- **타겟 환경**: Ubuntu 24.04, CLI (터미널)
- **개발 언어**: 순수 C 언어 (POSIX 시스템 콜 전면 활용)
- **핵심 로그라인**: "플레이어는 다크웹의 정보 브로커가 되어, 독점 기밀 문서를 선점하고 까다로운 조건의 NPC 고객에게 경쟁자보다 먼저 팔아넘겨 막대한 부를 축적해야 한다."

## 2. 6대 핵심 게임플레이 및 기능 (Core Functionalities)

### A. 다중 속성 기밀 독점 수집 및 NPC 의뢰 판매 (Tag-based NPC Trading)
- **단일 매물 선착순 독점 (Exclusive Rights)**: 시장의 모든 기밀 문서는 서버당 단 1개만 존재하는 독점 매물입니다. 유저가 `/buy`로 구매하는 즉시 상점에서 사라집니다.
- **다중 속성(Tags) 부여**: 모든 정보는 속성 태그(예: `[B기업]`, `[고객정보]`, `[군사무기]`)를 1~N개 지니고 있습니다.
- **글로벌 NPC 고객 의뢰 보드 (Race Condition Selling)**: 특정 속성 태그 조합(예: `[B기업]+[고객정보]`)을 완벽히 요구하며 랜덤한 현상금을 내건 NPC 고객들이 나타납니다.
- **서버 측 조합 검증 알고리즘**: 유저가 여러 문서를 묶어서 제출(`/sell`)했을 때, 서버는 제출한 문서들의 태그 합집합이 NPC가 요구하는 태그 '조합'을 완전히 포함(부분집합 관계)하는지 판별합니다. NPC가 요구하지 않은 불필요한 초과 태그가 포함되어 있어도 매각이 가능하지만, 해당 초과 태그가 담긴 문서도 함께 소모(소실)되므로 플레이어는 기회비용을 저울질해야 합니다.
- **선착순 매각 충돌**: 조건을 충족하는 기밀을 지닌 유저 중 가장 먼저 판매 명령(`/sell [내 기밀 ID 여러 개] [NPC ID]`)을 전송한 단 1명만이 현상금을 독식합니다.
- **정보 수익 포기 및 강제 폐기 (Trash Bin)**: 샌드박스가 꽉 차거나 가치가 없어진(상장 폐지된) 악성 재고가 쌓였을 때, `/dispose` 명령을 통해 **어떠한 비용 환급도 없이 0원에 파일을 강제 파기(`unlink`)**시켜 샌드박스 빈자리만 겨우 확보하는 뼈아픈 손절매를 해야 합니다.

### B. 비동기 이벤트 시스템 및 소셜 사보타주 (Events & Sabotage)
- **속보 뉴스 및 사전 힌트 (News & Hints)**: NPC가 등장하기 직전 혹은 규칙적인 타이밍마다, 서버의 이벤트 스레드가 전역 통신망에 '뉴스 속보'를 브로드캐스트합니다. (예: `[속보] 국방부, 신무기 도면 유출 의혹`). 유저들은 이를 단서로 삼아 조만간 `[군사무기]` 태그를 요구하는 VIP NPC가 출몰할 것을 미리 예측하고, 블랙마켓에서 해당 태그의 매물을 선제적으로 싹쓸이(매집)하는 고도의 눈치 플레이가 가능해집니다.
- **[공중파 유출] 상장 폐지 및 존버 메타 (Public Leak & Freeze)**: 특정 기밀이 민간 대중 언론에 퍼졌다는 속보가 터지면, 해당 태그는 약 3~5분간 **'동결(가치 0원 폭락)'** 처리됩니다. 동결 기간 동안 시장에 스폰되지 않으며 NPC도 받지 않는 최악의 악성 재고가 되어 유저의 인벤토리를 막아버립니다. 유저는 눈물을 머금고 0원에 강제 파쇄(`/dispose`)하여 손절할지, 아니면 동결 타이머 종료 후 뉴스가 잠잠해지며 **'기적의 재상장(가치 1.0배 복구)'**이 될 때까지 샌드박스의 한 칸 기회비용을 날리면서 버틸지(다이아몬드 핸드) 극한의 심리적 딜레마를 겪습니다.
- **의뢰 에이징(Aging) 스폰 시스템**: 특정 태그만 몰려서 수요와 공급이 맞지 않는 교착 상태를 막기 위해, 아무도 해결하지 못하고 오래 남은 NPC 의뢰는 이벤트 스레드에 의해 지속적으로 만료(Despawn) 처리되어 생태계 순환을 돕습니다.
- **시각적 글리치(Glitch) 공격**: 막대한 수수료를 내고 `/rumor` 공작을 띄우면, 다른 유저들의 화면에 떠 있는 NPC 보드의 특정 가격이나 태그 텍스트가 일시적으로 깨지거나 뒤섞이는 UI 글리치 연출이 가해져 상대방의 판단을 물리적으로 흐리게 합니다. 단, 무분별한 남용을 막기 위해 서버 내에 공유되는 쿨타임이 적용되어 연속적인 사용은 불가능합니다.

### C. 파일 시스템 기반 "인벤토리 샌드박스" (Inventory Sandbox)
- **실물 파일 이동 및 권한 제어**: 마스터 계층의 문서는 서버 측 단일 파일 시스템 통제 하에 해당 유저 전용 샌드박스 폴더로 물리적 이동(`rename`) 처리됩니다.
- **최대 보유 개수 제한 (Inventory Limit)**: 각 유저가 소유할 수 있는 파일(문서)의 개수는 서버 설정(예: 5칸)으로 엄격히 제한됩니다. 샌드박스가 가득 차면 추가 구매가 불가능하므로, 유저는 반드시 `/dispose`를 통해 가치가 없어진 악성 문서를 강제 파쇄(`unlink()`)하여 여유 공간을 스스로 관리해야 합니다.
- **인벤토리 용량 제한 (Capacity Limit)**: 플레이어가 기밀 정보를 구매하여 저장할 수 있는 인벤토리 샌드박스의 한도는 인당 최대 5개로 엄격하게 제한됩니다.

### D. 패닉 버튼 "긴급 파기 미니게임" 시그널 흐름 (Panic Escape Sequence)
- **경찰 레이드와 시그널 인과관계 명확화**: 
  1. 서버가 전체 유저에게 "경찰 추적망 활성화!" 패킷 전송.
  2. 클라이언트 렌더러가 화면을 붉게 점멸시킴.
  3. 놀란 유저가 1차 안전장치인 `Ctrl+C` 입력 (하드웨어 `SIGINT` 시그널 발생).
  4. 클라이언트 운영체제 수준 핸들러가 이를 가로채 `volatile sig_atomic_t is_panic = 1;` 세팅. (단, 시스템 먹통 방지를 위해 본래의 강제 종료 기능은 `Ctrl+\`인 `SIGQUIT` 시그널로 변경 매핑하여 비상시 터미널 복구 탈출 경로를 별도로 보장합니다.)
  5. 메인 루프가 플래그를 읽고 채팅/마켓을 강제 잠금한 뒤, "암구호 타이핑" 창을 띄우며 최후의 방탈출(초기화) 진행.

### E. 초간단 Key 기반 인증 및 유저 지속성 (Key Auth & DB Persistence)
- **Key(토큰) 대조 접속**: 복잡한 아이디/비밀번호나 가입 절차 없이, 플레이어는 고유한 'Key(문자열)' 단 하나만 입력하여 게임에 빠르고 직관적으로 접속합니다.
- **단일 파일 기반 구조체 I/O**: 모든 유저의 자산 데이터(구조체)는 `users.dat`라는 단 하나의 파일에 저장되며, 오프셋 단위 입출력으로 자산 영속성을 유지합니다. 신규 Key 입력 시 파일 끝에 새 구조체를 덧붙여(Append) 자동 가입 처리합니다.
- **논리적 계정 소각**: 파산한 유저의 계정은 파일을 물리적으로 삭제하는 대신 구조체 내 자산 수치를 특수 값(-1)으로 덮어씌워, 해당 Key의 재접속을 영구적으로 차단합니다.

### F. 승리 및 게임 오버 조건 (Win & Game Over Condition)
- **목표 금액 달성 및 자금 전송**: 서버 내 접속한 플레이어 중 가장 먼저 특정 목표 금액(할당량)을 달성하는 즉시, 해당 유저가 모아둔 모든 자금이 익명의 누군가에게 강제 전송(초기화)됩니다.
- **승리자 엔딩 (Win)**: 자금을 모두 빼앗긴 승리자의 화면에는 *"모든 빚을 갚았군. 약속대로 넌 이제 자유다."*라는 시스템 채팅이 출력되며, 게임에서 명예롭게 퇴장(승리)하게 됩니다.
- **패배 및 정산 라운드 (Post-Game Lobby & Final Countdown)**: 승리자가 발생하는 즉시 서버의 모든 거래와 마켓은 정지(Freeze) 상태가 되고, 유저들의 화면에는 화려한 글리치 효과와 함께 **[최종 브로커 정산 랭킹 (Scoreboard)]**이 렌더링됩니다. 서버는 30초간의 "에필로그 채팅 시간(자유 대화)"을 제공하여 플레이어들이 서로 마지막 소감을 나눌 수 있도록 하며, 30초 타이머가 끝나면 패배자들의 화면에 *"약속을 지키지 못했군. 남은 빚은 목숨으로 받지."*라는 섬뜩한 메시지가 출력되면서 터미널 화면을 깔끔하게 정상 복원(`endwin()`)한 뒤 안전하게 강제 종료됩니다.
- **파산 시 즉각 처형 및 계정 소각 (Bankruptcy & Permadeath)**: **자금(money)과 인벤토리가 동시에 0이 되어** 더 이상 아무런 행동도 취할 수 없는 완전한 깡통 상태가 된 플레이어는 즉시 게임 오버 처리되어 서버에서 퇴출됩니다. 자금이 0이더라도 인벤토리에 매각 가능한 문서가 남아 있다면 파산으로 간주하지 않습니다. 해당 계정 데이터는 서버 내에서 영구적으로 소각되어, 플레이어는 다른 계정으로 처음부터 다시 시작해야만 합니다.

## 3. 시스템 아키텍처 및 시스템 콜 활용 계획 (System Architecture)

### A. 네트워크 및 다중 접속 동기화 (Multithreading & Socket Lock)
- 멀티스레드 기반 TCP 소켓 서버 (Thread-per-client 모델).
- **[교착 상태(Deadlock) 원천 차단 룰]**: NPC 의뢰를 맞추기 위해 다수의 정보 파일(ID 3번, 5번 등)에 동시에 `pthread_mutex_lock`을 걸어야 할 경우, 반드시 **[가장 ID 숫자가 작은 파일 자원부터 오름차순으로 Lock을 획득하도록 강제]**하여 순환 대기(Circular Wait)에 의한 운영체제론적 데드락 에러를 미연에 원천 차단합니다.
- **시스템 콜**: `socket()`, `bind()`, `listen()`, `accept()`, `pthread_create()`, `pthread_mutex_lock/unlock()`

### B. 시그널 제어 및 스레드-안전 이벤트 타이머 (Signals & Timers)
- **이벤트 스레드**: 서버에서 별도 스레드가 `sleep()`에 기반해 Aging 처리.
- **시스템 콜**: `sleep()`, `alarm()`, `signal/sigaction()`, `kill()`

### C. 샌드박스 시스템 파일 마운트 무결성 룰 (FileSystem Restrictions)
- **[마운트 EXDEV 제약 요건 방어]**: `rename()` 호출 시 다른 파티션 대역이면 이동이 거부되는 OS 취약점을 피해, 마켓 마스터 폴더와 모든 유저의 개인 샌드박스는 반드시 부모 폴더를 공유하는 로컬 트리 구조(예: `./data/market` 과 `./data/users`) 내에 상대 경로로 묶여 있도록 설계합니다.
- **시스템 콜**: `mkdir()`, `rename()`, `link()`, `unlink()`, `opendir()`, `readdir()`, `stat()`, `chmod()`

### D. 유저 DB 관리 및 이중 접속 방지 (File I/O & In-Memory Check)
- `users.dat` 단일 파일에 대한 로우 레벨 구조체 단위 오프셋 파일 입출력을 적용하여 구현 복잡도를 대폭 낮춥니다.
- **인메모리 배열 기반 중복 접속 차단 및 하트비트 세션 동기화**: 무거운 OS 수준의 파일 락(`fcntl`) 대신, 서버 전역 변수로 현재 접속 중인 Key들을 저장하는 배열(`active_keys`)을 선언하여 인메모리 상에서 중복 로그인을 단순하고 빠르게 차단합니다. 또한, 워치독에 의한 자식 서버 재기동 시 세션 불일치를 예방하기 위해 서버는 주기적인 하트비트(Heartbeat) 신호를 클라이언트에 송출하며, 3회 이상 응답이 없는 클라이언트는 소켓 반환과 동시에 `active_keys` 및 `users.dat` 상의 로그인 활성화 플래그를 정교하게 자동 리셋(Reset)하도록 설계합니다.

### E. 워치독 서버 모니터 (Watchdog Server Monitor)
- **무중단 감시 체계**: 부모(Monitor) 프로세스가 `fork()`를 통해 자식(메인 TCP 게임 서버) 프로세스를 실행합니다. 자식 프로세스에 치명적 에러가 생겨 종료되더라도, 부모가 `waitpid()`로 상태를 감지하고 즉각 서버를 재가동하는 방호벽을 구축합니다.

## 4. 유저 인터페이스 (User Interface) 및 UI 렌더링 룰
- **최소 해상도 방어막 (Fail-Safe)**: 클라이언트 구동 시 `getmaxyx()`로 최소 요구 해상도(`100x30`) 미달을 감지하면, 직관적인 에러 문구(`현재 %dx%d / 목표 100x30`)를 출력하고 강제 종료하여 터미널 표준(80x24) 구동 환경 하의 UI 블록 겹침 에러(Segmentation Fault)를 선제적으로 차단합니다.
- **런타임 동적 리사이징 (`KEY_RESIZE`)**: 프로그램 실행 도중 유저가 창 크기를 변경하여 발생하는 `SIGWINCH` 인터럽트를 `wgetch() == KEY_RESIZE` 분기문으로 잡아내어, 메모리상의 기존 윈도우를 모두 해제(`delwin()`)하고 바뀐 해상도를 갱신해 전체 화면을 동적으로 다시 렌더링(Redraw)합니다.
- **비율 기반 분할과 잔여 공간 보정 (`newwin`)**: `COLS * 0.4` 등 실수 기반 비율 연산 시 C언어의 정수 버림(Truncation) 오차로 인해 화면 가장자리에 1~2칸의 검은 여백이 남는 현상을 방지합니다. 맨 우측이나 맨 하단에 위치하는 종착지 윈도우는 절대 비율이 아닌 **전체 길이에서 이전 창들의 폭을 뺀 나머지(잔여) 크기**를 할당하여 1픽셀의 빈틈 없이 터미널을 꽉 채웁니다.
- **입력 윈도우 분리 및 버퍼 오버플로우 방어 (Input Engine)**: 채팅/로그 출력 창과 프롬프트 입력창을 물리적으로 2개의 윈도우로 완전히 분리합니다. 출력창에만 `scrollok` 옵션을 주어 입력 커서가 밀려올라가는 UX 버그를 막고, 단 1줄 크기의 프롬프트 입력창엔 우측 끝에 '퀵 헬프(Quick Help)' 가이드를 하드코딩합니다. 레이아웃 붕괴 방지를 위해 유저 타건 문자열의 길이가 가용 너비를 초과하면 메모리 락을 걸고 입력을 무시하는 Buffer Overflow 방어 로직을 적용합니다.
- **수동 라인 버퍼링**: `cbreak()`, `timeout(0)`, `noecho()` 함수군을 조합해 비동기 타자를 지원.
- **POSIX 타임 동기화**: `<time.h>`의 `time()`, `localtime()`을 통해 서버 알림 수신 시 로컬 시각을 `[%02d:%02d:%02d]` 형태로 로깅하여 게임의 몰입도를 극대화합니다.
- **UI 사보타주 시각 이펙트**: `standout()` 및 컬러 뒤섞음을 통해 찌라시/해킹 피드백 직관적 제시.

## 5. 프로젝트 평가 기준 방어 및 체크리스트 (Evaluation Defense Checklist)
- Deadlock 방지 락 체인(순환대기 해결 오름차순 알고리즘 구현), EXDEV 회피 파일 마운트 제한, Async-Signal-Safety 로직 분기 등 상업적 규모에서도 가장 난해한 C언어/운영체제론적 크리티컬 에러들을 아키텍처 설계 단계부터 선제적으로 완벽 방어함.
